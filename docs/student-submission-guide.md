# 大作业提交与演示说明

本文档面向课程大作业验收，帮助老师快速了解项目完成度、运行方式、工程亮点和后续改进空间。

## 项目定位

本项目是一个校园众包任务平台，围绕校内互助场景提供账号注册登录、任务发布、任务接单、任务验收、纠纷处理、消息沟通、钱包流水、校园认证和管理后台等功能。

项目采用前后端分离架构：

- 前端：React、Vite、Tailwind CSS，适配桌面端和移动端。
- 后端：Spring Boot、Spring Security、Spring Data JPA。
- 数据库：MySQL 作为本地/部署运行库，H2 用于自动化测试。
- 打包：支持桌面应用打包和 Android 调试 APK 构建。

## 建议演示流程

1. 启动后端和前端，展示登录页、服务器地址配置和账号登录。
2. 注册普通用户，进入任务大厅，展示任务筛选、收藏和任务详情。
3. 使用一个账号发布任务，确认钱包余额预扣和流水记录。
4. 使用另一个账号接单，进入订单页，提交完成说明。
5. 切回发布者账号，验收任务，展示接单者收入到账和双方评价入口。
6. 展示任务消息会话，说明发布者和接单者只能访问相关任务会话。
7. 使用管理员账号进入后台，展示认证审核、用户管理、余额调整、权限管理和纠纷处理。
8. 展示自动化测试命令和构建脚本，说明项目不是只停留在页面展示。

## 已完成能力

- 账号体系：注册、登录、JWT 会话恢复、账号记忆、退出登录。
- 任务市场：分类、搜索、收藏、发布、接单、提交、验收、取消、纠纷。
- 钱包流水：发布任务预扣、任务完成结算、取消/纠纷退款、后台余额调整。
- 消息沟通：围绕任务建立会话，限制非参与者访问。
- 用户中心：资料编辑、头像更新、校园认证申请、历史记录、钱包明细。
- 管理后台：用户检索、封禁/解封、删除用户、认证审核、权限配置、纠纷裁定。
- 多端构建：浏览器运行、桌面便携包、Android 调试 APK。
- 自动化验证：后端业务测试、前端工具函数测试、ESLint 检查。

## 工程亮点

- 业务链路完整：任务从发布到结算形成闭环，不只是静态页面。
- 权限边界明确：普通用户、管理员和细分后台权限分离。
- 交易留痕：余额变化会写入流水，便于追踪问题。
- 组件拆分：前端按页面、布局、聊天、工具函数和 API 服务分层。
- 测试覆盖：后端覆盖认证、安全、用户自助、任务生命周期等核心场景。
- 发布意识：提供构建脚本、安卓构建说明、本地配置示例和生产安全保护。

## 运行与验收命令

后端测试：

```powershell
cd campus-backend
.\mvnw.cmd test
```

前端检查：

```powershell
cd campus-frontend
npm run test
npm run lint
npm run build
```

桌面打包：

```powershell
.\build-software.ps1
```

Android 调试 APK：

```powershell
.\build-android.ps1 -ApiBaseUrl "http://<后端可访问地址>:8080"
```

## 答辩时建议说明的不足

这些不足不影响作为课程大作业展示，但如果继续做成真实软件，需要继续优化：

- 钱包目前是平台内模拟余额，不应直接等同真实支付系统。
- 消息目前使用轮询，正式上线可升级为 WebSocket 或 SSE。
- 任务和用户之间仍有字符串关联，后续可改为数据库外键和更严格的数据模型。
- 生产部署需要配置真实 JWT 密钥、管理员密码、数据库账号和 HTTPS。
- 缺少正式截图、操作视频和线上演示地址，提交前建议补充。
